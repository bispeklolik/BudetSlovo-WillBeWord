# Спек: интерфейс «Записи» (дерево + карточки), рабочие окна, раздел «Конспекты»

_Дата: 2026-06-20. Контекст: пользователь сообщил, что кнопки переименования/перемещения
записи «не работают» (причина — `window.prompt` не поддерживается Electron), создавать папки
негде, и нет нормального интерфейса папок/проектов. Утверждён вид «дерево слева + карточки
справа». Дополнительно: отдельный раздел «Конспекты» для готовых выжимок._

## Проблема (баг)

`Home.tsx` использует `window.prompt` в `rename`, `move`, `renameFolder`. В Electron
`prompt()` — no-op (ничего не показывает, возвращает `undefined`), поэтому кнопки немые.
`confirm()`/`alert()` работают, поэтому замена/перераспознавание (на confirm) — работают.

## Решение

### 1. Внутренние окна вместо `window.prompt`
- **`InputModal`** — окно ввода строки: заголовок, подпись, начальное значение, Enter=OK,
  Esc=отмена. Используется для: создать папку, переименовать запись, переименовать папку,
  редактировать заголовок конспекта.
- **`MoveModal`** — выбор папки назначения из списка существующих + «В корень». Заменяет
  ручной ввод пути при перемещении записи.

### 2. Экран «Записи»: дерево слева + карточки справа
- **`Sidebar`** (дерево папок):
  - «Все записи» (корень) сверху; «Конспекты» — особый закреплённый узел внизу/сверху.
  - Папки вложенно (раскрытие/сворачивание); подсветка выбранной; число записей.
  - «+ Папка» — создаёт внутри выбранной папки.
  - На папке: переименовать; удалить (только если пустая — иначе подсказка «сначала
    освободите папку», чтобы не терять записи).
- **Правая панель** — записи выбранной папки красивыми карточками (как сейчас), кнопки
  карандаш→`InputModal` (переименование), папка→`MoveModal` (перемещение). Перетаскивание
  карточки на узел дерева сохраняется.
- **Пустые папки не исчезают:** список папок (включая пустые) хранится в настройках
  (`settings.folders: string[]`). Дерево = объединение этого списка и путей, выведенных из
  записей. Создание/переименование/удаление папки правит и список, и (для переименования)
  префиксы у записей (существующий IPC `project:renameFolder`).
- **Чистая логика дерева** `buildFolderTree(paths)` — выносится в `shared`, покрывается
  тестами.

### 3. Раздел «Конспекты» (сохранённые выжимки)
Отдельное хранилище текстовых заметок — не аудио.
- **Модель** `Note { id, title, body, kind: 'summary'|'thoughts'|'note', sourceSlug?,
  sourceTitle?, createdAt, updatedAt }`.
- **Хранилище** `notes.json` в `DATA_DIR` (массив; main — единственный писатель).
- **IPC** `notes:list` → `Note[]`; `notes:save` (без id → создаёт, с id → обновляет) → `Note`;
  `notes:delete(id)`. Preload + `api` + типы env + заглушка.
- **UI:** в `Sidebar` узел «Конспекты» → правая панель показывает карточки заметок (заголовок,
  бейдж типа, источник, дата; действия: открыть, копировать, экспорт .docx, удалить).
- **`NoteViewer`** — окно: редактируемые заголовок и тело (textarea) + Сохранить, Копировать,
  Скачать .docx (через существующий `export:textDocx`), Удалить, Закрыть.
- **Сохранение в конспекты:**
  - `SummaryPanel` → кнопка «Сохранить в конспекты» (`kind:'summary'`).
  - `HighlightsPanel` (список мыслей) → «Сохранить в конспекты» (`kind:'thoughts'`,
    тело — список мыслей с таймкодами).

## Порядок реализации (каждый шаг — сборка + коммит)
1. `InputModal` + `MoveModal`; переписать `rename`/`move`/`renameFolder`/создание папки в
   `Home` на них; персист `settings.folders`. **(чинит баг)**
2. `buildFolderTree` + тесты; `Sidebar`; переписать `Home` на две панели.
3. Хранилище заметок + IPC + preload/api/типы; узел «Конспекты»; карточки; `NoteViewer`;
   «Сохранить в конспекты» в `SummaryPanel` и `HighlightsPanel`.

## Принципы
- Никаких `window.prompt`. Все диалоги — внутренние компоненты.
- Записи (аудио + расшифровки) — ценные, удаление не вводим. Папки удаляем только пустые.
- Чистую логику (дерево, заметки-хелперы) — тестируем.
