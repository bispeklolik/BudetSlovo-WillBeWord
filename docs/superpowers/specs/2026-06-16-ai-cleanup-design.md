# Спека v1: ИИ-причёсывание текста («Слово»)

_2026-06-16. Первая спека ИИ-слоя. Пользователь одобрил поведение простыми словами (см. ниже «Что видит пользователь»); техрешения — на мне. Пользователь не вычитывает этот документ; его ревью = реакция на рабочий прототип._

## Цель

Кнопка в редакторе превращает «сырую» расшифровку в чистый читаемый текст **локально на машине пользователя**: убирает слова-паразиты и лишние повторы, приводит к аккуратному виду, **сохраняя манеру речи** говорящего; неразборчивые/плохо распознанные места **чинит по смыслу** из контекста. Все правки ИИ **видны отдельным цветом**, оригинал восстановим, аудио-синхрон не ломается.

## Что видит пользователь (одобренное поведение)

1. В редакторе кнопка «Причесать ИИ».
2. Нажатие → локальная обработка с прогрессом → текст становится аккуратнее; всё, что тронул ИИ, **подсвечено отдельным цветом**; наведение показывает оригинал; одно нажатие «Отменить правки ИИ» + обычный Ctrl+Z.
3. Первый раз: кнопка «Скачать ИИ-модель» (один раз, на D:).
4. Приватность: по умолчанию локально, явная пометка. Облако (Claude) — следующая спека.

## Архитектура

Новый модуль `src/main/ai/` (main — единственный писатель; ИИ-задача идёт через ту же очередь, что и расшифровка, чтобы не конкурировать за GPU).

**Интерфейс провайдера** `src/main/ai/provider.ts`:
```ts
export interface CleanupOptions { systemPrompt?: string }   // свой промт — опц., на будущее
export interface AiProvider {
  id: 'local-llama' | 'claude'
  name: string
  isLocal: boolean
  isAvailable(): Promise<boolean>          // модель скачана / сервер жив / ключ задан
  cleanupTurn(text: string, uncertain: string[], opts: CleanupOptions): Promise<string>  // вернуть причёсанный текст реплики
}
```
- **`LocalLlamaProvider`** (v1): спавнит встроенный `llama-server.exe` (CUDA) на `127.0.0.1:<порт>`, шлёт OpenAI-совместимые запросы. Тот же паттерн «подпроцесс + локальный сервер», что и `faster-whisper-xxl.exe`. Лениво стартует при первом ИИ-вызове, гасится по простою/выходу.
- **`ClaudeProvider`** — следующая спека (Anthropic API, ключ пользователя, метка «облако»).
- Реестр провайдеров + выбор в настройках; путь к модели; (позже) ключ Claude и свой промт.

**Модель:** по умолчанию **Qwen2.5-7B-Instruct** (GGUF Q4_K_M, ~4.7 ГБ — комфортно влезает в 12 ГБ RTX 3060, сильный русский). Альтернатива на заметку: Vikhr/Saiga (русско-тюненые). Качается по требованию в `D:\Apps\slovo-data\models\`. Движок (`llama-server.exe` + .dll) — в сборке (`resources/llama/`), CUDA-сборка; CPU/Vulkan-фолбэк на машинах без NVIDIA.

## Как чистим и как красим правки (тех. сердце)

Не просим модель про id слов (локальные 7–8B это делают ненадёжно):
1. Обработка **по реплике** (turn = слова с id и таймкодами).
2. Промт: системный (убери паразиты/повторы; приведи к читаемому виду; **сохрани стиль говорящего**; неразборчивое **почини по контексту**; не выдумывай факты) + текст реплики, где **низко-уверенные слова помечены** (напр. `слово?`) — чтобы ремонт целил туда.
3. Модель возвращает **причёсанный текст реплики** (её сильная сторона).
4. **Детерминированный пословный diff** оригинал↔причёсано (LCS) → патчи поверх существующего оверлея: неизменённые слова сохраняют id+таймкод (караоке живёт); заменённые → `t` + `t0`(оригинал) + `src:'ai'`; удалённые паразиты — вон; вставленные — без таймкода (караоке пропускает, как ручные вставки).
5. **Цвет:** у слова поле `src:'ai'` → отдельный цвет (сине-бирюзовый, отличный от жёлтого/красного уверенности и ручных правок); тултип «ИИ поправил · исходно: …». Места «ремонта неразборчивого» (оригинал был низко-уверен И ИИ поменял) — насыщеннее.

**Модель данных:** слову добавляется `src?: 'ai'` (ручные правки — без `src`, как сейчас). Реюз `t0`. В `project.json` — флаг, что реплика причёсана (для revert и идемпотентного повторного запуска). «Отменить правки ИИ» = снять `ai`-патчи, вернуть `t0`.

## Длинные записи

ИИ-задача в очереди (конкуренция 1, как расшифровка). Идём по репликам с прогрессом «N из M реплик». Очень длинные реплики (>~1500 токенов) бьём по предложениям. Можно свернуть окно — идёт в фоне; отмена как у расшифровки (`job.log`, живучесть при Ctrl+R).

## Ошибки / граничные случаи

- Модель не скачана → кнопка «Скачать ИИ-модель» (прогресс загрузки), действие задизейблено.
- `llama-server` не стартовал / нет CUDA → понятная подсказка + CPU-фолбэк (медленнее).
- Модель вернула мусор/слишком расходящийся текст (diff меняет >X% или ломает смысл по длине) → пометить реплику «не удалось причесать», оставить оригинал, не падать.
- Идемпотентность: повторный «Причесать» не накладывается дважды (работаем от оригинала `t0`).

## Тестирование / приёмка

- Юнит: LCS-diff оригинал↔причёсано → корректные патчи (замена/удаление/вставка), сохранение id+таймкодов у неизменённых.
- Юнит: парсинг ответа модели, защита от мусора (порог расхождения).
- Интеграция/спайк (де-риск, делается ПЕРВЫМ): поднять локальную модель, прогнать реальную реплику из `novaya-zapis-34`, глазами оценить: паразиты убраны, стиль сохранён, неясное починено разумно. Если качество слабое — пересмотреть модель/промт ДО UI.
- Ручная приёмка в приложении: кнопка → прогресс → цветные правки → тултип оригинала → отмена → экспорт.
