# Спек: ИИ-обезличивание расшифровки

_Дата: 2026-06-20. Утверждён вид «переключатель в редакторе». Цель: получить обезличенную
версию транскрипта (без имён/мест/организаций/идентифицирующих деталей), пригодную для отправки
в облако или публикации в блоге. Обезличивание работает на ЛОКАЛЬНОЙ модели — иначе сырые данные
уже ушли бы в облако._

## Принципы
- **Локально.** Обезличивание считает локальная модель (Ollama qwen2.5). Облако не участвует.
- **Оригинал неизменен.** Замены — слой поверх (как пометки «Причесать ИИ»). Тумблер переключает
  только отображение.
- **Честно про границы.** ИИ ловит почти всё, но не 100%. Поэтому: подсветка замен + строка-
  напоминание + ручная правка списка. Финальный взгляд за пользователем.

## Модель данных
- `AnonRule = { find: string; replace: string; kind: 'name'|'place'|'org'|'other' }`.
- `ProjectMeta.anon?: AnonRule[]` — список правил замены, хранится в project.json.
- `find` — точная подстрока КАК В ТЕКСТЕ; каждая словоформа отдельно («Ева», «Евы», «Евой»),
  все формы одного человека → один и тот же `replace`.

## Чистая логика (shared, с тестами)
- `buildAnonOverlay(turns, rules): Map<wordId, string>` — для каждого правила находит подряд идущие
  слова, чьи нормализованные формы совпадают с `find` (без регистра/пунктуации); первому слову
  спана ставит `replace`, остальным — `''` (скрыть). Длинные `find` применяются раньше коротких.
  Один источник правды для отображения И экспорта.

## ИИ (локальный провайдер)
- `anonymize(text): Promise<AnonRule[]>` — находит людей (все формы), места, организации,
  прочие явные идентификаторы; подбирает нейтральные замены («Клиент»/«Клиентка»/«Имя N»;
  «город»/«вуз»; «организация»). `format:'json'`, один вызов на весь транскрипт, `num_ctx` 32768,
  `num_predict` ~2000. Обычные слова не трогает.
- Джоб `runAnonymize(slug)`: ensureOllama → собрать текст → anonymize → сохранить `meta.anon`.

## IPC / preload / api
- `ai:anonymize(slug)` → запускает джоб, возвращает meta.
- `ai:setAnon(slug, rules)` → сохраняет отредактированный список (ручная правка/добавление).
- `export:run` получает доп. флаг `anon` — экспортировать обезличенную версию.

## UI
- **Меню «✨ ИИ» → «Обезличить»** (запуск). Когда `meta.anon` есть: тумблер + «Список замен».
- **Тумблер «Обезличено»** в шапке редактора (виден при наличии правил). ON → транскрипт рисует
  замены (`overlay`), подсвеченные классом `.word.anon`; скрытые хвосты спанов не рисуются.
  Клик по слову/караоке/аудио работают (замена наследует тайминг исходного слова).
- **Баннер** в режиме ON: «Подсвечено то, что заменил ИИ. Мог что-то пропустить — пробегись глазами».
- **`AnonPanel`** (список замен): правила `find → replace` (kind-бейдж), удаление правила, поле
  «добавить пропущенное слово» (find + replace). Сохраняет через `ai:setAnon`.
- **Экспорт/копирование уважают тумблер**: ON → выгружается/копируется обезличенный текст
  (через `buildAnonOverlay`). Это и есть путь в облако/блог.

## Порядок реализации (сборка + тесты + коммит на каждом шаге)
1. shared: `AnonRule` + `buildAnonOverlay` + тесты.
2. ИИ: `anonymize` в провайдере + `runAnonymize` + IPC `ai:anonymize`/`ai:setAnon` + preload/api/типы.
3. Редактор: тумблер + overlay + рендер в `TranscriptView` (подсветка, скрытие хвостов) + баннер + «Обезличить» в меню ИИ.
4. `AnonPanel`: просмотр/удаление/добавление правил.
5. Экспорт/копирование уважают тумблер.

## Границы (честно)
- Косвенные приметы («единственный, кто…») ИИ ловит хуже прямых имён — отсюда подсветка и ручная правка.
- Русские падежи: модель возвращает каждую встретившуюся форму; в обезличенном тексте возможна
  лёгкая грамматическая шероховатость («с Клиентка») — это допустимо, цель — безопасность, не литература.
- Очень длинные сессии: один вызов на 32k контекста; если хвост не попал — видно по неподсвеченному
  имени, добавляется вручную.
